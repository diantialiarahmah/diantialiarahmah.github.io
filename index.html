Tentang Smart Incubator Monitoring System

Smart Incubator Monitoring System adalah sistem pemantauan cerdas untuk mengawasi kualitas suhu dan kelembapan inkubator bayi secara real-time menggunakan Arduino Uno, sensor DHT11, dan buzzer.

Sensor DHT11 membaca kondisi lingkungan di dalam inkubator dan data diproses oleh Arduino Uno. Apabila suhu atau kelembapan berada di luar rentang aman, sistem akan mengaktifkan buzzer sebagai alarm peringatan dini.

Sistem ini dirancang sebagai solusi sederhana, ekonomis, dan andal yang dapat dikembangkan lebih lanjut ke sistem monitoring terpusat berbasis ROS 2 dalam ekosistem Smart Hospital.

Pentingnya Smart Incubator Monitoring System

Pengaturan Suhu dan Kelembapan

Bayi, terutama bayi prematur, belum mampu mengatur suhu tubuhnya sendiri. Inkubator berfungsi menjaga suhu dan kelembapan tetap stabil agar bayi aman dan nyaman.

Risiko Kondisi Tidak Stabil

Kondisi inkubator yang tidak stabil dapat menyebabkan gangguan pernapasan, iritasi kulit, kehilangan cairan tubuh, hingga meningkatkan risiko infeksi.

Keterbatasan Pemantauan Manual

Pemantauan manual bersifat berkala dan sangat bergantung pada ketelitian manusia sehingga berisiko terjadi keterlambatan penanganan.

Sistem Otomatis Berbasis Mikrokontroler

Sistem otomatis memungkinkan pemantauan terus-menerus dan memberikan peringatan dini secara cepat melalui alarm.

Fitur Sistem

  • Real-Time Temperature & Humidity Monitoring
  • Automatic Alarm System (Buzzer Alert)
  • Centralized Monitoring (Multi-Inkubator Ready)
  • ROS 2 Publisher‚ÄìSubscriber Integration

      Code arduino ide:

      #define DHTPIN 2        // Pin DATA DHT11
#define DHTTYPE DHT11
#define BUZZER 8        // Pin buzzer

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  dht.begin();
  pinMode(BUZZER, OUTPUT);

  Serial.println("=== Monitoring Inkubator Bayi ===");
}

void loop() {
  float kelembapan = dht.readHumidity();
  float suhu = dht.readTemperature(); // Celcius

  // Cek jika sensor gagal membaca
  if (isnan(suhu) || isnan(kelembapan)) {
    Serial.println("Gagal membaca sensor DHT11!");
    digitalWrite(BUZZER, HIGH);
    delay(1000);
    digitalWrite(BUZZER, LOW);
    delay(1000);
    return;
  }

  // Tampilkan data
  Serial.print("Suhu: ");
  Serial.print(suhu);
  Serial.print(" ¬∞C | Kelembapan: ");
  Serial.print(kelembapan);
  Serial.println(" %");

  // Logika alarm inkubator bayi
  if (suhu < 32 || suhu > 36 || kelembapan < 50 || kelembapan > 70) {
    Serial.println("‚ö†Ô∏è PERINGATAN: Kondisi TIDAK AMAN!");
    digitalWrite(BUZZER, HIGH);
  } else {
    Serial.println("‚úÖ Kondisi AMAN");
    digitalWrite(BUZZER, LOW);
  }

  Serial.println("----------------------------");
  delay(2000);
}

    import rclpy
from rclpy.node import Node
from std_msgs.msg import Float32, String, Bool

class IncubatorLogicNode(Node):

    def __init__(self):
        super().__init__('incubator_logic_node')

        # Threshold (batas aman)
        self.temp_min = 32.0
        self.temp_max = 36.0
        self.hum_min = 40.0
        self.hum_max = 60.0

        self.temperature = None
        self.humidity = None

        # Subscriber
        self.create_subscription(
            Float32,
            '/incubator/temperature',
            self.temperature_callback,
            10
        )

        self.create_subscription(
            Float32,
            '/incubator/humidity',
            self.humidity_callback,
            10
        )

        # Publisher
        self.status_pub = self.create_publisher(String, '/incubator/status', 10)
        self.alarm_pub = self.create_publisher(Bool, '/incubator/alarm', 10)

        self.get_logger().info("Incubator Logic Node started")

    def temperature_callback(self, msg):
        self.temperature = msg.data
        self.evaluate_condition()

    def humidity_callback(self, msg):
        self.humidity = msg.data
        self.evaluate_condition()

    def evaluate_condition(self):
        if self.temperature is None or self.humidity is None:
            return

        status_msg = String()
        alarm_msg = Bool()

        if (self.temperature < self.temp_min or self.temperature > self.temp_max or
            self.humidity < self.hum_min or self.humidity > self.hum_max):

            status_msg.data = "DANGER"
            alarm_msg.data = True

        else:
            status_msg.data = "NORMAL"
            alarm_msg.data = False

        self.status_pub.publish(status_msg)
        self.alarm_pub.publish(alarm_msg)

        self.get_logger().info(
            f"Temp: {self.temperature} | Hum: {self.humidity} | Status: {status_msg.data}"
        )

def main(args=None):
    rclpy.init(args=args)
    node = IncubatorLogicNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

üõ†Ô∏è Troubleshooting & Problem Solving

‚ö†Ô∏è Kendala Umum & Solusi Teknis

Tahap ini bertujuan untuk mengidentifikasi dan menangani permasalahan yang muncul selama proses pengembangan Smart Incubator Monitoring System, khususnya pada integrasi Arduino/ESP32 dengan ROS 2, sehingga sistem dapat berjalan secara stabil dan optimal.

‚úÖ Tested Solutions ‚Äì 5 Common Issues

üî¥ Problem #1 ‚Äì Arduino Gagal Dikompilasi

Penyebab:
Library belum terpasang dengan benar atau pemilihan board tidak sesuai.

Solusi:

  • Pastikan board Arduino/ESP32 sudah dipilih dengan benar pada Arduino IDE
  • Instal library sensor (DHT) dan dependensi ROS 2 / micro-ROS yang diperlukan

üî¥ Problem #2 ‚Äì Node ROS 2 Tidak Terhubung ke Mikrokontroler

Penyebab:
Koneksi serial atau konfigurasi komunikasi ROS 2 tidak sesuai.

Solusi:

  • Periksa port serial yang digunakan
  • Pastikan micro-ROS agent berjalan dan menggunakan port yang benar

üî¥ Problem #3 ‚Äì Port Komunikasi ROS 2 Terblokir Firewall

Penyebab:
Firewall sistem operasi memblokir port UDP yang digunakan ROS 2.

Solusi:

  • Tambahkan aturan firewall untuk membuka port komunikasi ROS 2
  • Nonaktifkan firewall sementara saat pengujian

üî¥ Problem #4 ‚Äì ROS 2 Daemon Tidak Merespons (Timeout)

Penyebab:
ROS 2 daemon gagal berjalan akibat konflik proses atau konfigurasi.

Solusi:

  • Restart ROS 2 daemon menggunakan perintah ros2 daemon stop dan ros2 daemon start
  • Pastikan environment ROS 2 sudah di-source dengan benar

üî¥ Problem #5 ‚Äì Perintah ROS 2 Tidak Dikenali

Penyebab:
Environment ROS 2 belum diaktifkan pada terminal.

Solusi:

  • Jalankan perintah source /opt/ros/<distro>/setup.bash
  • Tambahkan perintah source ke file .bashrc agar otomatis
